This is the documentation of Sierra - A lightweight Python3 framework for building and integrating web applications. Sierra is a Python3 library for building and integrating web applications with HTML and CSS using simple enough syntax. You can develop your web applications purely with Python, taking advantage of its functionalities and integrating them to the fullest.

The documentation will elaborate on the syntax for this. In future releases, integration with Flask will be available, and support might be extended for brython over JS . Using this library will create an 'index.html' and a 'style.css' in the same directory, and index.html can be viewed from there.

First, let us import the library

from sierra import *

If Sierra is manually being cloned from the repository and setup.py not being used, do make sure to run pip install -r requirements.txt

Once imported, the first two lines of syntax are mandatory. 

    
    if __name__ == "__main__":
        title('Your page title')

    # title(Title, icon='False') takes two arguments:
    # Title(str, compulsory)   : Title of the HTML file.
    # icon(str, optional)      : Tab icon to be displayed. Should be a .ico file. Defaults to no icon.

head() is used to add a header to your page, and it takes multiple arguments for styling.
It is recommended to only use arg 'type' or arg 'font_size' at once. Using both defaults to ignoring arg type

The body of your web application which contains the main content can be started with openBody() , and it can be styled using optional arguments.

openBody(background='False', background_color='white', background_image=False, opacity=False, background_size='cover', background_attachment='fixed', background_position=False, background_repeat=False)

It can similarly be closed using closeBody() which takes no arguments.

Initialization of CSS can be done using addInitc() and it is not mandatory. On a different note, any color code works with CSS arguments. HEX, RGB or anything else.

addInitc(box_sizing='False', margin=False, padding=False, border=False, position='relative')

Font links can be added using addFont(font_link) 


    addFont(font_link="https://fonts.googleapis.com/css2?family=Roboto+Mono:wght@300&display=swap")

    # addFont() takes in one argument, which is the font link. It is the href to the font specified.
    # If the font is being taken from Google Fonts, copy only the href.
    # For example if the embed for the font 'Roboto' is <link href="https://fonts.googleapis.com/css2
    # ?family=Roboto&display=swap" rel="stylesheet">, copy only the href on it, which is
    # "https://fonts.googleapis.com/css2?family=Roboto&display=swap".
    # Specify the font family as "'Roboto', sans-serif" (only in double quotes, since there are single 
    # quotes inside) in the argument font_family in the CSS function

Tags - Open and close

Multiple tags can be opened and closed simply by using openTags() and closeTags(). closeTagBefore() can be used to close any tag before the opening of another. This is intended to rectify unclosed tags is an easier manner. 

    
    openTags(any_tag, *args)
    # openTags('tag1', 'tag2', 'tag3')
    closeTags(any_tag, *args)
    # closeTags('tag2', 'tag3')
    closeTagBefore(tag_to_close, tag_to_close_before)
    # closeTagBefore('tag1', 'tag3')

Using division, section and p tags

Division, section and paragraph tags can be used with class tTags() , tTags() standing for 'titleTags', since the division and section tags basically stand for titles to identify areas within a page.
tTags() takes in three parameters. With the exception of the p tag, both the division and the section tags need to be closed manually. However, the p tag can be closed manually if desired so by simply leaving the second argument on object start_p blank, and auto closing of that tag defaults to bool False. Entering bool True closes the tag immediately after text is entered. 

    
    tTags(p=False, div_class='False', sec_class='False')

    # p is a bool value, the name of the class is given to <div> and <section> in the second and third arguments respectively, if required

    a = tTags(p=True)
    a.start_p('Hello World!', close=True)
    a.css(color='#6e560f', font_family="Times New Roman", font_size="10px")

    # a.start_p(text, close=bool(False)) is used to start the paragraph and takes in one mandatory argument 'text', which is the paragraph content.  
    # Arg close, if True, closes the paragraph once text is entered. Arg 'close' is set to False by default

CSS automatically applies to the tag whose argument has been mentioned in tTags(). Multiple tags can be opened at once in any combination too but if this is done, CSS automatically applies the same arguments in a.css() to all tTags being used. It is hence recommended to open tags one at a time.
Once the required tags are opened, a.start_div(takes no args) or a.start_sec(takes no args) or a.start_p() is used to open the tag mentioned in tTags(). 


    a = tTags(div_class='newClass')
    a.start_div()                   # Takes no argument
    a.css(color='rgb(61, 37, 49)', font_family="Arial")

         # ---Content of the division---

    closeTags('div')                 # Close <div>

    a = tTags(sec_class='anotherClass')
    a.start_sec()                    # Takes no argument
    a.css(color='rgb(38, 62, 89)', font_family="Times New Roman")

         # ---Content of the section---

    closeTags('section')              # Close <section>

Adding images

Images can be added using class addImg() which takes in four arguments: src , href , alt and img_class , arg src being mandatory, as it is the location of the image. href if used, links the image to any specified location on click. alt is the alternate text that is displayed if for some reason, the image is not displayed. img_class is the class of the image, which is recommended if there are multiple images and multiple combinations of styling are needed. 


    # a = addImg(src:str, href="False", alt="This is an image", img_class='False')        # Defaults
    a = addImg("location/to/the/image.jpeg", href="https://google.com", alt="Some image", img_class="newImg")

    a.show()                    # Takes no argument and displays the image with the conditions given

    a.css(height='30px', width='20px')

    # It is recommended to use arg img_class if multiple combinations of styling is used for multiple images

Adding a table (pandas supported)

Adding a table could never have been easier with class startTable() which takes no arguments and has objects createTable and getTable() .
Sierra is compatible with Pandas, meaning just enter enter the path to the CSV file in arg dataframe in getTable() , and the table will display! 

    
    a = startTable()
    a.getTable("path/to/file.csv")

A table can also be added using createTable() . It takes in two arguments: cols:list and rows , where rows is a list of lists. cols is a list that contains the header row, which is basically the title of each column. rows is a list of lists that contains the rows to be entered 

    
    a = startTable()
    c = ['england', 'best', 'eight', 'euros']
    r1 = ['kane', 'grealish', 'sancho', 'foden']
    r2 = ['mount', 'rice', 'chilwell', 'stones']
    r = [r1, r2]
    a.createTable(cols=c, rows=r)

    # Outputs table:

    # england   best       eight     euros

    # kane      grealish   sancho    foden
    # mount     rice       chilwell  stones

Of course, CSS can be added with a.css() 

    
    a.css(border=False, width=False, height=False, border_collapse=False, color='black',
    font_family="Arial", font_weight=False, text_align=False, font_size=False, margin=False,
    background_color='white')

    # Do keep in mind that any color code (eg. RGB, HEX) can be used to add color
Working with Bullet lists

Bullet lists can be passed in as regular lists on Sierra to display on the web application using addBullets() It takes in one mandatory argument and four optional arguments. The first argument points takes in a list of points to add and adds them in. 


    addBullets(points, ul=False, start="1", type="False", inner=False)

    # points(list, compulsory): Takes in a list of bullets to add
    # ul(bool, optional)      : If set to True, it adds bullets (unordered) rather than an ordered list
    # start(str, optional)    : Starts the ordered list from a particular starting position
    # type(str, optional)     : Sets the type of ordered/unordered list
    # inner(bool, optional)   : Determines if bullet lists have to be added inside of a bullet list item. False by default
          

    # closeTags('ol' OR 'ul') has to be done for every inner list added. That is, for every addBullet(inner=True), 
    # a closeTags() must be used. Here', I've added one inner list and hence just one closeTags() will suffice.
    # If ul is set to True for an inner list, it must be closeTags('ul'). If False (default), closeTags('ol') 
    # It isn't needed if inner is set to False (default)

    
    a = ['this', 'is', 'a', 'list']
    addBullets(points=a, inner=True, ul=True)
    addBullets(points=a, inner=False, type='i')
    closeTags('ul') 

    # Output:
  • this
  • is
  • a
  • list
    1. this
    2. is
    3. a
    4. list

    As you can see, an inner list is created with type 'i'
    You can even set start to '5', and the inner list starts with a Roman 'v'
Other functions

Python variables can be loaded onto the webpage easily with the use of f-strings within writeHTML or var.start_p or anything, thus providing ground for Sierra as an integrated library that is not just confined to the walls of HTML and CSS

These are a list of other functions that can be used

    href(link:str, text:str)
    # Links any text to a specified location

    code(codeblock:str)
    # Shows any given text inside as code

    writeHTML(text)

    writeCSS(tag, *args)

    # marquee {

    #  background-color: rgb(240, 237, 228);
    #  }
    
    var b
    # The variable 'b' can be used to break a sentence into the next line.
